# 产品需求文档 (PRD)：审批中心

**版本**：V1.0  
**日期**：2026-06-11  
**状态**：初版（HTML 原型）

## 1. 文档概述

### 1.1 背景

平台需对高风险 Agent/Skill 执行、Workspace 内资产发布及 Skillhub 集市上架进行统一审批。审批中心按**审批状态**组织主视图，支持三类审批类型筛选与差异化详情展示。

### 1.2 三类审批

| 类型 ID | 中文名 | 触发场景 | 通过后 |
|---------|--------|----------|--------|
| `l3_exec` | L3级操作 | L3 风险等级 Agent/Skill 操作需审批拦截 | Run 继续 |
| `platform_publish` | 平台发布 | Agent/Skill 提交发布到 Workspace | 资源已发布 |
| `skillhub_publish` | Skillhub 发布 | Skill 提交上架 Skillhub 集市 | Skill 上架 |

### 1.3 目标用户

- Workspace / 平台管理员（审批人）
- Agent/Skill 开发者（发起人，「我发起的」Tab）

## 3. 页面结构

**主 Tab（状态优先）**

1. **待我审批** — `assignedTo = 当前用户` 且 `status = pending`
2. **审批历史** — `status ∈ {approved, rejected}`
3. **我发起的** — `submitter = 当前用户`

**工具栏**：审批类型、Workspace、搜索、Show/Hide Filters

## 6. F-203 AI 总结

### 6.1 场景

| 角色 | 入口 | 说明 |
|------|------|------|
| 提交人 | 「我发起的」Tab → **提交前 AI 预审** | 提交审批前对草稿做风险自查 |
| 审批人 | 待我审批 / 历史 / 我发起的 → 详情抽屉顶部 | 审批前查看 AI 总结辅助决策 |
| 查看者 | 同上 | 只读查看历史项风险回顾 |

### 6.2 输出内容

- **风险等级**：高 / 中 / 低 + 综合风险分（0–100）
- **主要风险**：按审批类型（L3级操作 / 平台发布 / Skillhub 发布）规则化识别
- **建议**：区分提交人（自查整改）与审批人（复核关注点）
- **完整分析**：可展开查看 Markdown 详情

### 6.3 与 AI 预审报告区别

| 能力 | AI 预审报告 | AI 总结 |
|------|-------------|---------|
| 时机 | 发布类提交时自动生成 | 提交前可选预审 + 抽屉内始终展示 |
| 对象 | 主要覆盖平台/Skillhub 发布 | 三类审批均支持 |
| 视角 | 合规/内容审核结论 | 风险点 + 角色化行动建议 |
