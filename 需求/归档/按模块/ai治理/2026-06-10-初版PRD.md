# 产品需求文档 (PRD)：AI 治理

**版本**：V1.0  
**日期**：2026-06-10  
**状态**：初版（HTML 原型）

## 1. 文档概述

### 1.1 项目背景

企业在全面采用 AI Agent、Skill、MCP 等能力后，管理者需要跨 Workspace 审计用户行为与工具调用，识别风险并获取治理建议。AI 治理模块与 AI 观测（技术可观测性）互补，侧重合规、权限与使用规范。

### 1.2 产品目标

1. **全平台审计**：管理者可查看可见 Workspace 内的用户操作与工具执行记录。
2. **风险可见**：聚合高风险事件（导出、权限变更、MCP 失败等）。
3. **智能治理**：基于审计数据提供 AI 洞察与可执行建议。

### 1.3 目标用户

- **Workspace / 平台管理员**：审计、合规、权限治理。
- **安全合规人员**：导出行为、API Key、权限变更追溯。

### 1.4 数据范围

管理者可见的全部 Workspace；工具栏支持 **All Workspaces** 与单 Workspace 筛选。

## 2. 核心数据模型

| 实体 | 描述 | 核心属性 |
|------|------|----------|
| **AuditLog** | 用户操作审计 | action, user_id, resource_type, resource_id/name, detail_json, ip, created_at, workspace, risk |
| **ToolExecution** | 工具运行时执行 | type, resource, operation, duration_ms, status, trace_id, input, output, error, workspace |

参考 API（产品 spec §4.15）：

```
GET /api/v1/audit-logs?start&end&action&user_id&page
GET /api/v1/audit-logs/export
```

## 4. 字段明细

### 4.1 F-201 用户操作列表（8 列）

| 英文列名 | 中文列名 | 字段 | 说明 |
|------|------|------|------|
| Time | 时间 | created_at | 操作时间 |
| User | 用户 | user_id | 操作人 |
| Action | 操作类型 | action | login, agent.create, export, permission.change… |
| Resource Type | 资源类型 | resource_type | agent / skill / mcp / workflow / knowledge… |
| Resource | 资源 | resource | 资源名称或 ID |
| Workspace | 工作空间 | workspace | 所属 Workspace |
| Risk | 风险等级 | risk | low / medium / high |
| IP | 来源 IP | ip | 来源 IP |

**抽屉隐藏字段**：detail_json（完整变更详情）、trace_id（若有关联运行）

### 4.2 F-301 工具执行列表（9 列）

| 英文列名 | 中文列名 | 字段 | 说明 |
|------|------|------|------|
| Time | 时间 | time | 执行时间 |
| User | 用户 | user | 触发用户 |
| Type | 类型 | type | Agent / Skill / MCP / Tool |
| Resource | 资源 | resource | 智能体名 / Skill 名 / MCP Server / Tool 名 |
| Operation | 操作 | operation | run.trigger, tool.invoke, mcp.call, skill.execute… |
| Duration | 耗时 | duration_ms | 耗时（ms） |
| Status | 状态 | status | success / failed / timeout |
| Workspace | 工作空间 | workspace | 所属 Workspace |
| Trace | 链路 | trace_id | 关联 Trace，可跳转 AI 观测 |

**抽屉扩展字段**：input、output、error、agent、workflow

## 5. 与 AI 观测的关系

| 模块 | 侧重 |
|------|------|
| AI 观测 | Trace 性能、成本、延迟、Observation 树 |
| AI 治理 | 谁做了什么、合规风险、治理策略建议 |

两者通过 trace_id 关联；原型阶段 Trace 跳转为 alert 占位。
