# 产品需求文档 (PRD)：运营分析 — AI 智能运营

> **版本**：V1.0  
> **日期**：2026-06-11  
> **关联原型**：`原型/运营/运营分析/index.html`  
> **父文档**：[`2026-06-11-初版PRD.md`](./2026-06-11-初版PRD.md)

## 1. 概述

AI 智能运营在运营分析模块提供「读指标 + 给建议」能力，侧重**业务影响与运营动作**，与 AI观测（技术根因）、AI治理（合规根因）互补。

## 2. 能力列表

| ID | 名称 | 入口 | 优先级 |
|----|------|------|--------|
| AI-101 | AI 智能运营 | 平台总览顶部面板 | P1 |
| AI-102 | 异常自动解读 | 需关注表格行点击 | P1 |
| AI-103 | 看板内追问 | 各 Tab 右下角浮动按钮 | P2 |
| AI-104 | 运营周报 | 总览工具栏「生成周报」 | P2 |

## 3. AI-101 默认洞察

**输入**：空 query + 当前 Workspace + time_range  
**输出**：

- title：智能运营概览
- tags：成本优化、用户增长、资产运营、流程效率、质量体验（子集）
- content：权限范围、本周 KPI 摘要、3 条建议
- citedMetrics：引用的 KPI 名称数组

## 4. AI-101 追问意图（原型规则）

| 关键词 | 标题 | 标签 |
|--------|------|------|
| 成本/费用/花钱 | 成本分析 | 成本优化 |
| 用户/活跃/DAU | 用户增长分析 | 用户增长 |
| 资产/Agent/Skill/闲置 | 资产运营分析 | 资产运营 |
| 审批/发布/积压 | 审批效率分析 | 流程效率 |
| 错误/质量/满意度 | 质量体验分析 | 质量体验 |

## 5. AI-102 异常解读

点击「需关注」行，展示一行 AI 摘要：根因（mock 规则匹配 category）+ 一条建议。

## 6. AI-103 看板内追问

浮动按钮打开小型对话框，注入 `{ tab, workspace, timeRange }` 上下文；总览/使用/成本 Tab 有专属规则，其余 Tab 通用 fallback。

## 7. AI-104 运营周报

生成 Markdown 预览（alert 或 modal），含 KPI 表格化摘要与 5 条建议。

## 8. 非功能

- 分析范围 = 用户可见 Workspace，与工具栏筛选一致
- 原型为规则引擎；生产对接 LLM + OpsSnapshot API
