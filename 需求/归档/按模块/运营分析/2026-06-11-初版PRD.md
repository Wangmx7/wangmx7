# 产品需求文档 (PRD)：运营分析

**版本**：V1.0  
**日期**：2026-06-11  
**状态**：初版（HTML 原型）

## 1. 文档概述

### 1.1 背景

xSparkOps 已具备 AI观测（技术可观测）、AI治理（合规审计）、审批中心（流程管控）等模块，但缺少**业务运营视角**的跨域指标聚合与解读能力。运营分析模块面向平台/Workspace 负责人、产品运营与财务运维，回答「平台用得怎么样、值不值、瓶颈在哪」。

### 1.2 产品目标

1. **一屏总览**：跨 Workspace 健康度、使用、成本、审批、质量关键指标。
2. **分域看板**：使用、资产、成本、审批、集市、质量六域深度分析。
3. **AI 智能运营**：默认洞察 + 中文追问，输出可执行运营建议。
4. **聚合不重复**：明细排查跳转 AI观测 / AI治理 / 审批中心，不在本模块复刻 Trace/审计列表。

### 1.3 目标用户

| 角色 | 典型诉求 |
|------|----------|
| 平台 / Workspace 负责人 | 平台健康度、成本趋势、审批积压 |
| 产品 / 运营 | DAU、功能 adoption、资产复用率 |
| 财务 / 运维 | 成本分布、预算使用率、降本机会 |

### 1.4 与 sibling 模块边界

| 模块 | 视角 | 运营分析中的角色 |
|------|------|------------------|
| AI观测 | 技术 Trace/延迟/Model | 聚合成本/错误，跳转明细 |
| AI治理 | 合规审计 | 聚合活跃用户、风险事件 |
| 审批中心 | 流程 | 审批漏斗与 SLA 专看板 |

## 2. 核心数据模型（聚合层）

```
OpsSnapshot(workspace_id?, time_range) → {
  overview: { active_users, sessions, agent_calls, skill_calls, total_cost, ... },
  usage: { dau, wau, mau, funnel[], top_users[], top_tags[] },
  assets: { counts, distribution[], top_assets[], idle_trend[] },
  cost: { total, tokens, by_model[], by_workspace[], by_agent[] },
  approval: { pending, avg_wait, by_type[], backlog[], recent[] },
  hub: { listings, views, downloads, top_skills[] },
  quality: { error_rate, feedback_count, satisfaction, top_errors[], feedbacks[] }
}
```

---

## 3. 信息架构

**Tab（7 个）**：平台总览、使用分析、资产效能、成本效率、审批发布、集市运营、质量反馈

**全局工具栏**：Workspace 下拉、时间范围（7d/30d/90d）、对比模式（环比占位）、导出报告（占位）
