## 1. 背景与目标

### 1.1 背景

xSparkOps 资产域需支持智能体（Agent）的全生命周期管理：创建、调试、提交发布审批、正式运行与执行追溯。

**参考产品**：[Clawith](https://www.clawith.ai/zh-Hans/docs/) 创建 Agent 的核心交互为 **5 步向导**：基础信息 → 灵魂/性格（`soul.md`）→ 技能配置 → 权限（L1 自动 / L2 通知 / L3 审批）→ 渠道/发布绑定。

**平台现状**：

- 审批中心已支持 `platform_publish` 类型（Agent/Skill 发布到 Workspace）
- AI 观测已有 Trace 列表，可作为执行记录的外链下钻目标

### 1.2 目标

-  **5 步全屏创建向导**
- Step 5 **提交审批**后 Agent 进入**待审批**，审批通过后才**已发布**
- 未发布状态可在 **Playground 调试中心**调试，正式对外运行需已发布
- 提供 Agent **执行记录**，Trace 可跳转 **AI 观测**
- 与审批中心端到端 mock 联动可验收

## 2. 用户与场景

| 角色 | 场景 |
|------|------|
| Agent 开发者 | 全屏向导创建/编辑 Agent、Playground 调试、提交发布审批、查看执行记录 |
| Workspace 管理员 | 在审批中心处理 `platform_publish · Agent` 审批单，查看配置摘要 |
| 运维/分析人员 | 查看 Agent 执行记录，通过 Trace 下钻 AI 观测 |

## 3. 功能需求

### 3.1 Agent 状态机

| 状态 | 中文 | 说明 | 可编辑 | 可运行 |
|------|------|------|--------|--------|
| `draft` | 草稿 | 向导保存，未提交 | 是 | 仅 Playground |
| `pending_approval` | 待审批 | Step 5 提交后，已生成审批单 | 否（只读） | 仅 Playground |
| `published` | 已发布 | 审批中心 **通过** 后 | 否（需新版本） | 是（含对外正式运行） |
| `rejected` | 已驳回 | 审批中心 **拒绝** 后 | 是（改后重提） | 仅 Playground |

**关键规则**：

- Step 5 点击「提交审批」→ Agent 状态变为 `pending_approval`，**不会**直接变为已发布
- 审批中心对对应 `platform_publish` 单执行 **通过** → Agent 变为 `published`，写入 `publishedAt`、`publishedVersion`
- 审批 **驳回** → Agent 变为 `rejected`，保留 `approvalId` 与驳回备注，允许编辑后再次提交
- **Playground** 在草稿 / 待审批 / 已驳回均可调试；对话、工作流等对外正式运行仅 **已发布** 可用

### 3.2 列表页

**路径**：`原型/资产/智能体/index.html`

**工具栏**：

- 搜索：Agent 名称 / slug
- 状态筛选：草稿 / 待审批 / 已发布 / 已驳回
- 主操作：创建 Agent（进入全屏向导）

**卡片展示**：

| 元素 | 说明 |
|------|------|
| 状态 Tag | `draft` / `pending_approval` / `published` / `rejected` |
| 名称 + slug | 已发布显示版本号 |
| 摘要 | 两行截断 |
| 元信息 | 待审批显示审批单 ID；已驳回显示驳回原因摘要 |

**操作按状态差异化**：

| 状态 | 卡片操作 |
|------|----------|
| 草稿 | 编辑、提交审批、执行记录、Playground、克隆 |
| 待审批 | 查看配置（只读）、查看审批、执行记录、Playground、克隆 |
| 已发布 | 查看、执行记录、Playground、克隆 |
| 已驳回 | 编辑、提交审批、执行记录、Playground、克隆 |

点击卡片打开**详情抽屉**（配置摘要 / 执行记录 / Playground 三个 Tab）。

### 3.3 全屏创建向导

创建/编辑采用**全屏创建页**（非弹窗），布局为：

- **顶栏**：返回列表、标题、保存草稿
- **左侧步骤导航**：5 步纵向 Tab（序号 + 标题 + 说明；当前高亮、已完成打勾）
- **右侧表单区**：当前步骤内容 + 底部上一步 / 下一步 / 提交审批

**Step 1 基础信息**

- **先输入名称**，标识（slug）根据名称**自动生成**（英文转连字符；中文常用字拼音映射）
- 标识展示为 `agent/{slug}` 前缀，用户**可手动编辑**；手动编辑后不再随名称自动更新
- 另含：摘要、分类、图标

**Step 2 灵魂与 Prompt**

- 模型、温度（滑块）、系统提示词（必填）、启动指令、输出要求

**Step 3 能力授权**

- Skill / Tool / MCP 多选
- **知识库**多选（展示名称、描述、文档数）

**Step 4 风险与审批**

- 自主级别：L1 自动执行 / L2 通知确认 / L3 审批拦截（卡片单选）
- 沙箱策略：卡片单选，每项含标签、一句话摘要与**详细说明**

| 沙箱策略 | 标签 | 适用说明 |
|----------|------|----------|
| 只读沙箱 | 最安全 | 隔离环境，所有 Tool/MCP 强制只读，无外部写副作用 |
| Workspace 内读写 | 推荐 | 仅可读写当前 Workspace 内资源，不可跨空间 |
| 受限外部 API | 需审核 | 白名单外部 API，有 QPS/数据量上限与审计 |
| 生产只读 | 运维常用 | 可连生产只读数据源诊断，写操作禁用，L3 仍需审批 |

**Step 5 发布预览**

- 发布范围、版本号、变更说明、用途、风险边界
- 发布预览摘要（名称、slug、模型、能力、自主级别）
- 按钮：**保存草稿** | **提交审批**（非「直接发布」）
- 提交成功 Toast：「已提交审批，状态：待审批」+ 链接审批中心

**只读预览模式**：`pending_approval` / `published` 状态下打开向导为只读，展示五步配置摘要。

### 3.4 详情抽屉

| Tab | 内容 | 可见性 |
|-----|------|--------|
| 配置摘要 | 五步配置只读摘要（模型、Prompt、能力、自主级别、发布范围等） | 各状态 |
| 执行记录 | Run 列表，降序 | 各状态 |
| Playground | 调试中心；未发布顶部提示条说明调试与正式运行区别 | 各状态均可调试 |

### 3.5 执行记录

**列表字段**：

| 列 | 字段 | 说明 |
|----|------|------|
| 开始时间 | `startedAt` | 降序默认 |
| 触发来源 | `trigger` | 见下表 |
| 输入摘要 | `inputSummary` | 截断展示 |
| 状态 | `status` | `running` / `succeeded` / `failed` / `waiting_approval` |
| 耗时 | `durationMs` | 毫秒 |
| Token | `tokens` | 可选 |
| Trace | `traceId` | 链接至 `原型/运营/ai观测/index.html?traceId=xxx` |

**触发来源 `trigger`**：

| 值 | 说明 |
|----|------|
| `playground` | Playground 调试（未发布亦可产生） |
| `chat` | 对话触发（通常需已发布） |
| `workflow` | 工作流触发 |
| `scheduled` | 定时任务 |
| `waiting_approval` | L3 敏感操作被审批拦截（与审批中心 `l3_exec` 概念呼应） |

**写入时机**：

- 预置 mock：每个 Agent 3–5 条历史 Run
- Playground 运行成功 → `ApprovalBridge.appendRunRecord()` 追加 `trigger: playground`
- 已发布样例含 `chat` / `workflow`；L3 Agent 样例含 `waiting_approval`

### 3.6 审批联动

#### 3.6.1 触发与流转

```
智能体 Step5/列表「提交审批」
  → submitForApproval()
  → 创建 platform_publish 审批单
  → Agent.status = pending_approval

审批中心「通过」
  → resolveApproval(id, { approved: true })
  → Agent.status = published，写入 publishedAt / publishedVersion

审批中心「驳回」
  → resolveApproval(id, { approved: false, comment })
  → Agent.status = rejected，保留 rejectComment
  → 开发者编辑后可再次 submitForApproval()
```

#### 3.6.2 审批单字段（`platform_publish`）

| 字段 | 值 / 说明 |
|------|-----------|
| `type` | `platform_publish` |
| `resourceType` | `Agent` |
| `resource` | Agent slug |
| `version` | 待发布版本号（如 `1.0.0`） |
| `submitter` | 当前用户 |
| `workspace` | 发布 Workspace |
| `detail.publishScope` | 发布范围 |
| `detail.changeLog` | 变更说明 |
| `detail.purpose` | 用途说明 |
| `detail.riskBoundary` | 风险边界 |
| `detail.configSummary` | 五步配置摘要（模型、能力、自主级别等） |
| `detail.aiReview` | AI 预审摘要（mock） |

#### 3.6.3 跨模块同步（`共享/mock/approval-bridge.js`）

原型使用 **localStorage** 在智能体页与审批中心页之间同步：

| 方法 | 说明 |
|------|------|
| `loadAgents()` | 读取 Agent 列表（seed + 用户创建 + 状态覆盖） |
| `saveAgent(agent)` | 保存草稿 |
| `submitForApproval(agent)` | 创建审批单；Agent → `pending_approval`；返回 `approvalId` |
| `loadApprovals(staticItems)` | 合并静态 mock 与动态审批单 |
| `resolveApproval(id, { approved, comment })` | 通过/驳回；同步更新关联 Agent |
| `appendRunRecord(agentId, run)` | 写入执行记录 |
| `getRunRecords(agentId)` | 读取某 Agent 的全部 Run |
| `buildConfigSummary(agent)` | 生成五步能力摘要字符串 |

**存储键（原型）**：`xs-agents-v1`、`xs-approvals-dynamic-v1`、`xs-agent-runs-v1`

#### 3.6.4 审批中心侧改造

- 初始化：`APPROVAL_MOCK.items = ApprovalBridge.loadApprovals(...)`
- 通过/驳回：调用 `resolveApproval()` 持久化，而非仅改内存
- Agent 类型 `platform_publish` 抽屉增加 **配置摘要** 区块
- 抽屉内提供「查看智能体配置 →」跳转链接

#### 3.6.5 智能体页侧表现

- `pending_approval`：卡片「待审批」Tag + 「查看审批」跳转审批中心
- `published`：卡片显示 `publishedVersion`
- `rejected`：卡片展示驳回原因摘要 + 「编辑」重提

## 7. 验收标准

### 7.1 创建与状态

- [ ] 创建页为全屏布局，左侧 5 步导航、右侧表单

- [ ] 提交后审批中心「待我审批 / 我发起的」出现 **平台发布 · Agent** 条目

- [ ] 审批**驳回**后，Agent 变为**已驳回**，显示备注，可编辑并重新提交
